## Motivação

A auditoria (feita por sub-agente `Explore` no início da sessão) identificou 12 code smells. Os 5 mais impactantes foram resolvidos:

1. **Duplicação de parsing de `.env`** em 4 arquivos (`sync.py`, `bootstrap_supabase.py`, `weekly_digest.py`, `health_check.py`) — cada cópia tratava comentários/aspas diferente.
2. **Paths Windows hardcoded** como fallback em 4 scripts, contradizendo o fix F11 de `sync.py`.
3. **Queries SQL de agregação repetidas** em 3 arquivos com variações sutis de `GROUP BY`/`ORDER BY`.
4. **`weekly_digest.py` monolítico** (411 LOC misturando query, cloud, formatação e delivery) sem testes.
5. **Sem tooling declarativo**: `pyproject.toml` com 17 linhas, sem `pytest`/`ruff`/`mypy`/`pre-commit` configurados.

## Refatoração em 7 ondas

### Onda 1 — Módulo `geo_finops/config.py` (fonte única)

Novo módulo centralizando TODA resolução de filesystem e env vars:

```python
from geo_finops.config import (
    get_config_dir,      # ~/.config/geo-finops (respeita XDG)
    get_db_path,         # calls.db
    get_logs_dir,        # logs/
    get_digests_dir,     # digests/
    get_snapshot_path,   # landing-page-geo/public/finops-snapshot.json
    get_workspace_root,  # pai do repo
    parse_env_file,      # robusto: comentarios, aspas, encoding errors
    candidate_env_files, # cascata de 5 fontes, sem hardcode
    load_env_chain,      # merge de todas as fontes
    load_supabase_creds, # (url, key) ou (None, None)
    load_whatsapp_creds, # (token, phone_id)
    FLY_USD_PER_MONTH, VERCEL_USD_PER_MONTH, ALERT_DELTA_PCT,  # constantes
    DEFAULT_WHATSAPP_OWNER, CARAMASCHI_FINOPS_URL,
)
```

Novas env vars para override:

- `GEO_FINOPS_CONFIG_DIR` — diretório base
- `GEO_FINOPS_DB_PATH` — path completo do SQLite
- `GEO_FINOPS_ENV_FILE` — override explícito de `.env`
- `GEO_FINOPS_SNAPSHOT_PATH` — destino do JSON público
- `GEO_FINOPS_FLY_USD_MONTH`, `GEO_FINOPS_VERCEL_USD_MONTH` — custos cloud
- `GEO_FINOPS_ALERT_DELTA_PCT` — limiar de alerta no digest
- `GEO_FINOPS_WHATSAPP_OWNER`, `GEO_FINOPS_CARAMASCHI_URL` — endpoints

Os aliases privados `_candidate_env_files`, `_parse_env_file`, `_load_supabase_creds` permanecem em `sync.py` como re-exports para não quebrar os 14 testes existentes.

### Onda 2 — Módulo `geo_finops/aggregates.py` (SQL reutilizável)

Extração de todas as queries de agregação para um módulo testável:

```python
from geo_finops.aggregates import (
    totals,              # dict com calls, cost, tokens, period_start/end
    aggregate_by,        # por provider/project/model_id/task_type
    top_models,          # top N incluindo provider
    daily_timeseries,    # série diária dos últimos N dias
    top_hotspots,        # (project x model x task_type) top N
    sync_status_counts,  # pending/synced/error
)
```

Pattern: todas aceitam `conn` opcional para reuso em batches e usam a mesma gramática `start`/`end` ISO 8601. Elimina 3 cópias divergentes que existiam em `tracker.py`, `export_snapshot.py` e `aggregate_dashboard.py`.

### Onda 3 — Refatoração dos scripts para consumir `config` + `aggregates`

Reescrita de 4 scripts:

- `scripts/bootstrap_supabase.py` — usa `load_supabase_creds`, −30% LOC
- `scripts/export_snapshot.py` — usa `aggregates.totals/aggregate_by/top_models/daily_timeseries`, path de saída resolvido via `get_snapshot_path`, −25% LOC
- `scripts/aggregate_dashboard.py` — paths via `get_workspace_root`, URL caramaschi configurável, API de helpers privados preservada para os 12 testes existentes
- `scripts/health_check.py` — `Check` virou `@dataclass`, adapters/callers detectados via workspace canônico (pulam graciosamente se workspace ausente, permitindo CI em hosts genéricos)

Resultado: **zero paths Windows hardcoded** em qualquer script. `grep -r 'C:/Sandyboxclaude' scripts/` = 0 hits.

### Onda 4 — Pacote `geo_finops/digest/` (split do monolito)

`scripts/weekly_digest.py` de 411 LOC virou um thin CLI de 60 LOC. A lógica foi dividida em 4 módulos testáveis em isolamento:

```
geo_finops/digest/
├── __init__.py        # API publica agregada
├── cloud.py           # Fly.io + Vercel + GH Actions estimators
├── builders.py        # build_digest + week_window (ISO)
├── formatters.py      # format_markdown + format_whatsapp + format_json
└── reporters.py       # save_snapshot + send_whatsapp (Meta Graph v22)
```

Cada módulo tem dependências mínimas. `cloud.py` não conhece `digest`/`SQL`; `formatters.py` é puro (dict → str); `reporters.py` isola I/O (disco + HTTP). Isso permitiu escrever 20+ testes unitários vs 0 antes.
